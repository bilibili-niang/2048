# 2048小游戏需求开发文档

## 1. 项目概述

### 1.1 项目背景
本项目是一个基于Flutter框架开发的2048小游戏，主要面向Android平台。2048是一款经典的数字合并益智游戏，玩家通过滑动屏幕控制方块移动，相同数字的方块碰撞时会合并成一个更大的数字，目标是在有限空间内合成2048方块。

### 1.2 项目目标
- 开发一款流畅、美观的2048游戏
- 实现优美的动画效果，提升用户体验
- 集成震动反馈功能，增强交互体验
- 支持关卡选择和关卡记录持久化保存
- 支持深色/浅色模式适配
- 支持分数记录和本地数据持久化
- 支持Electron桌面端（Windows平台）

### 1.3 目标平台
- 主平台：Android（API 21+）
- 桌面平台：Windows（通过Electron）
- 兼容平台：iOS（可选）

## 4. 技术方案

### 4.1 技术选型

| 分类 | 技术 | 版本 | 说明 |
| :--- | :--- | :--- | :--- |
| 框架 | Flutter | 3.10.0+ | 跨平台UI框架 |
| 状态管理 | Provider | 6.0.5 | 状态管理库 |
| 震动 | vibration | 1.7.6 | 设备震动功能（移动端） |
| 存储 | shared_preferences | 2.2.2 | 本地数据存储 |
| 主题 | flutter/material | 内置 | 深色/浅色主题支持 |
| 桌面端 | Electron | 28.0.0+ | 桌面应用框架（Windows） |
| 桌面端 | flutter-web | 内置 | Flutter Web支持 |

### 4.2 架构设计

采用分层架构设计，主要分为以下层次：

- **UI层**：负责界面展示和用户交互
- **业务逻辑层**：游戏核心逻辑处理
- **数据层**：本地数据存储和读取
- **服务层**：震动反馈、主题管理等功能服务

#### 6.3.2 Electron主进程配置

`electron/main.js` 主要配置：

```javascript
const { app, BrowserWindow, Menu } = require('electron');
const path = require('path');

function createWindow() {
    const mainWindow = new BrowserWindow({
        width: 450,
        height: 600,
        minWidth: 360,
        minHeight: 500,
        icon: path.join(__dirname, 'resources', 'icon.ico'),
        webPreferences: {
            preload: path.join(__dirname, 'preload.js'),
            nodeIntegration: false,
            contextIsolation: true
        }
    });

    mainWindow.loadFile('build/web/index.html');
    
    // 创建菜单
    const menu = Menu.buildFromTemplate([
        {
            label: '游戏',
            submenu: [
                { label: '新游戏', accelerator: 'Ctrl+N', click: () => mainWindow.webContents.send('newGame') },
                { label: '重置', accelerator: 'Ctrl+R', click: () => mainWindow.webContents.send('resetGame') },
                { type: 'separator' },
                { label: '退出', accelerator: 'Ctrl+Q', click: () => app.quit() }
            ]
        },
        {
            label: '帮助',
            submenu: [
                { label: '关于', click: () => mainWindow.webContents.send('showAbout') }
            ]
        }
    ]);
    Menu.setApplicationMenu(menu);
}

app.whenReady().then(() => {
    createWindow();
    app.on('activate', () => {
        if (BrowserWindow.getAllWindows().length === 0) createWindow();
    });
});

app.on('window-all-closed', () => {
    if (process.platform !== 'darwin') app.quit();
});
```

## 7. 代码安全性

### 7.1 安全注意事项

| 风险点 | 风险等级 | 关联模块 | 说明 |
| :--- | :--- | :--- | :--- |
| 数据存储安全 | 低 | SharedPreferences | 存储的分数等数据无敏感信息 |
| 外部输入验证 | 低 | 游戏逻辑 | 游戏输入仅来自用户手势，无需额外验证 |
| 权限滥用 | 中 | 震动服务 | 震动功能需要用户授权，且提供开关控制 |
| 代码注入 | 低 | 所有模块 | 纯客户端应用，无网络请求 |

### 7.2 安全措施

1. **权限管理**: 震动功能在使用前检查权限状态
2. **边界检查**: 游戏逻辑中对所有数组访问进行边界检查，防止越界异常
3. **异常处理**: 在关键流程中添加异常捕获，确保应用稳定性
4. **数据验证**: 从SharedPreferences读取数据时进行类型验证和范围检查
